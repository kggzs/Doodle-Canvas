# Doodle-Canvas 审计报告

> 更新时间：2026-06-20
> 说明：本文件替代早期“纯前端 SPA”审计结论，记录当前后端化后的安全与架构状态。

## 总体结论

项目已完成关键安全架构迁移：第三方 API Key 不再存储于浏览器，模型调用统一走后端代理，生成结果转存到服务端文件库，项目数据保存到 MySQL。早期审计中最高风险的“API Key 明文 localStorage”和“任意动态代理 SSRF”问题已通过后端化路线消除。

当前主要风险已经从“前端暴露密钥”转为“后台配置正确性、上游错误可观测性、商业化和内容治理闭环”。

## 已修复或已缓解问题

| 编号 | 早期问题 | 当前状态 |
| --- | --- | --- |
| S-1 | API Key 明文存储于 localStorage | 已改为服务端 `model_channels.api_key` AES-GCM 加密存储 |
| S-2 | 动态代理可 SSRF | 已取消前端任意代理；生成结果远程转存有 URL 和 DNS 防护 |
| P-1 | localStorage 项目数据易丢失 | 项目已保存到服务端 `projects.canvas_data` |
| B-1 | IndexedDB 图片缓存无限制 | 上传和生成文件进入服务端 `files`，浏览器本地大对象保存前清理 |
| B-2 | 生成结果临时外链失效 | 生成结果必须转存本地；转存失败则生成失败并退款 |
| A-1 | 模型配置分散 | 后台三类模型页统一维护模型、渠道、计费和线路 |
| A-5 | Provider 逻辑散落 | 后端 `generation.js` 统一适配 Provider 请求和响应 |

## 当前安全措施

- Access/Refresh Token 类型隔离。
- API Key 加密存储，接口响应不返回明文或密文。
- `/api` 全局限流。
- 生产密钥强度校验。
- 文件访问查库校验 active 状态。
- 上传和转存文件使用魔数识别真实 MIME。
- 远程文件下载阻断内网、回环、链路本地和组播地址。
- 上游 401/403、超时、网络异常写入错误日志。
- 前端错误文案脱敏，后台保留原始排查信息。
- 已有历史生成记录的模型/渠道不硬删，保留外键追溯。

## 剩余风险与建议

### A-01 高优先级：审计日志未完整落库

当前有 `request_id`、运行日志和 `error_logs`，但管理员操作审计、关键业务访问日志还不完整。

建议：

- 新增 `audit_logs` 和 `access_logs`。
- 对管理员创建/修改/删除模型、渠道、计费、用户状态等操作写审计。
- 后台增加审计日志页面。

### A-02 高优先级：充值和卡密链路未实现

金币流水和扣费已具备，但充值订单、卡密兑换和对账报告还缺失。

建议：

- 增加 `recharge_orders`、`redeem_cards`。
- 用户侧增加兑换接口和余额页。
- 管理侧增加卡密批量生成、禁用、过期和订单查询。

### A-03 中优先级：内容治理未闭环

生成记录可以追溯，但审核、举报、风控规则、封禁流水未落地。

建议：

- 增加内容审核队列。
- 用户举报入口和后台处理页。
- 风控规则和事件表。
- 独立封禁历史，避免只依赖 `users.status`。

### A-04 中优先级：渠道熔断可继续增强

当前有请求数、成功/失败数、`circuitOpen` 字段和手动重置接口，但自动熔断恢复策略可继续完善。

建议：

- 定时健康检查。
- 半开探测。
- 按错误率自动熔断和恢复。

### A-05 低优先级：缺少专用测试脚本

当前主要依赖 `node --check`、构建、浏览器回归和手工 API 冒烟。

建议：

- 增加后端服务层单元测试。
- 增加生成适配器 mock upstream 集成测试。
- 增加关键页面 Playwright 冒烟。

## 依赖审计

历史验证中已使用官方 npm registry 执行：

```bash
npm audit --omit=dev --registry=https://registry.npmjs.org
npm --prefix server audit --omit=dev --registry=https://registry.npmjs.org
```

生产依赖漏洞数为 0。使用 `npmmirror` 时 audit 可能返回 404，审计应固定官方 registry。

## 结论

当前代码已经跨过最危险的密钥暴露阶段，核心生成链路具备可追溯、可扣费、可退款、可查错的闭环。下一阶段应优先补商业化、审计和内容治理，而不是继续扩展更多前端生成入口。
