# Doodle-Canvas 开发指南

这份文档给代码助手和协作者使用，记录当前项目结构、运行方式和容易踩坑的边界。

## 项目概况

Doodle-Canvas 当前不是纯前端项目。它由 Vue 前端和 Express 后端组成，后端负责账号、模型渠道、API Key 加密、生成代理、金币计费、文件存储、项目持久化和管理后台 API。

## 技术栈

- 前端：Vue 3、Vite、Vue Router、Pinia、Naive UI、Tailwind CSS、Vue Flow。
- 后端：Express、Sequelize、MySQL、Redis、Multer、Sharp、Winston。
- 部署：`npm run build` 生成 `dist/`，后端单服务托管前端和 API。

## 开发原则

- 前端不得保存第三方 API Key；模型调用统一走后端 `/api/generate/*` 和 `/api/chat/*`。
- 用户侧模型列表只使用 `/api/models`，不要回退到旧的前端内置模型作为可调用模型。
- 模型、渠道、绑定必须保持同一 `model_type`：`chat`、`image` 或 `video`。
- 删除已有历史生成记录的模型或渠道时，服务层会解绑并停用，不硬删。
- 生成结果必须落入本地存储和 `files` 表；远程 URL 转存失败应失败并退款。
- `/storage/*` 必须经过后端数据库状态校验，不要重新加裸 `express.static(server/storage)`。
- 生产环境必须有有效 `JWT_SECRET` 和 `AES_SECRET_KEY`。开发环境自动生成到 `server/.runtime.env`。
- 图片模型后端有最低 1 金币兜底，避免误配置为免费。

## 前端注意事项

- 路由根路径 `/` 是官网首页，登录后默认进入 `/projects`。
- `/admin` 默认进入 `/admin/dashboard`。
- `/admin/channels` 和 `/admin/billing` 已收敛跳转到 `/admin/models/chat`。
- 画布项目数据以服务端 `projects.canvas_data` 为准，保存前会清理 `base64`、`blob:`、`data:`、`upload://` 等浏览器本地数据。
- 生成完成或失败后应触发余额刷新事件，避免右上角积分过期。

## 后端注意事项

- API 成功响应使用 `success()`，错误响应使用 `error()`，保留 `request_id`。
- 鉴权接口只接受 access token；refresh token 只用于刷新。
- API Key 存储在 `model_channels.api_key`，接口响应只返回 `apiKeyConfigured` 和 `apiKeyValid`。
- 远程文件下载要经过安全 URL 校验、安全 DNS lookup、MIME 魔数识别和大小限制。
- 限流只挂在 `/api`，静态资源不能计入 API 限流。
- Redis 异常时，限流和黑名单有降级策略；生产鉴权默认 `AUTH_REDIS_FAILURE_MODE=deny`。
