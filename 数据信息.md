# 本地数据配置示例

不要在文档中记录真实密码、API Key、SMTP 密码或生产密钥。真实配置请放在 `server/.env`，并确保该文件不提交到版本库。

## MySQL

```env
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=canvas
DB_USER=canvas
DB_PASS=replace_with_mysql_password
```

初始化：

```bash
mysql -u root -p < server/sql/init.sql
```

## Redis

```env
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_PASS=
```

## 运行密钥

开发环境可留空，服务会自动生成到 `server/.runtime.env`：

```env
JWT_SECRET=
AES_SECRET_KEY=
```

生产环境建议显式配置：

```env
JWT_SECRET=replace_with_at_least_32_bytes_random_secret
AES_SECRET_KEY=replace_with_exactly_32_ascii_chars
```

## 文件存储

```env
STORAGE_ROOT=server/storage
STORAGE_BASE_URL=/storage
```

生产环境建议把 `STORAGE_ROOT` 指到 Git 工作区外，例如：

```env
STORAGE_ROOT=/www/wwwroot/doodle-canvas-storage
```
