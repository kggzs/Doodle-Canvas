## 1. 全局通用改造

### 1.1 AppHeader 组件改造 (`src/components/AppHeader.vue`)

**现状：** 当前 `AppHeader` 是全局复用的头部组件，右侧区域依次显示：
- "我的画布" 按钮
- "积分 {balance}" 按钮
- 用户名下拉菜单（用户中心 / 我的画布 / 退出登录）
- GitHub 图标
- 主题切换按钮

**改造后右侧区域顺序（从左到右）：**

| 序号 | 元素 | 说明 |
|------|------|------|
| 1 | 公告滚动条 | 长条滚动显示最新公告，点击弹出公告详情弹窗 |
| 2 | 模型价格 | 点击跳转到模型价格页面/弹窗 |
| 3 | 积分（使用记录） | 只显示积分使用记录，不显示余额 |
| 4 | 用户名下拉菜单 | 菜单项改为：用户中心、修改密码、退出登录（去掉「我的画布」） |
| 5 | 用户组标签 | 当前用户所属组，带颜色显示 |
| 6 | GitHub 链接 | 保留不变 |
| 7 | 黑夜/白天切换 | 保留不变 |

**实现要点：**
- 公告组件：新建 `AnnouncementBanner.vue` 子组件，嵌入 AppHeader 右侧，通过 SSE/轮询加载最新公告
- 积分按钮：将当前积分余额按钮改为"积分使用记录"入口，点击直接滚动或跳转
- 用户组标签：从 `currentUser.value?.userGroups` 获取，`currentUser` 是从 `stores/auth.js` 导出的
- 用户下拉菜单：将 `userOptions` 中的"我的画布"改为"修改密码"

### 1.2 新增公告数据结构

|------|------|----------|
| 原密码 | password | 必填 |

| 确认新密码 | password | 必须与新密码一致 |

### 4.3 后端 API

当前后端已有 `PUT /api/auth/password` 接口（`server/src/routes/auth.js:289-323`），传入 `{ oldPassword, newPassword }`。

修改成功后后端返回 `{ message: '密码修改成功' }`，并调用 `RefreshToken.revokeByUserId(userId, currentSessionId)` 撤销除当前会话外的所有 refresh_tokens。

前端在修改密码成功后：
1. 调用 `AuthService.changePassword()` 成功后
2. 调用 `logout()` 清除本地认证数据
3. 显示成功消息"密码修改成功，请重新登录"
4. 跳转到登录页 `/login`

## 10. 模型生成错误中文翻译

### 10.1 问题描述

模型生成时，上游 AI 提供商（如 OpenAI）可能返回：
```
The content you provided or machine outputted is blocked.
```
这个错误是英文的，需要翻译成中文返回给用户。

### 10.2 现状分析

错误处理流程：
1. 上游 API 返回错误 → `callUpstream()` 在 `server/src/services/generation.js:423-457` 处理
2. 上游错误信息通过 `errorMessageFromUpstream()` 提取 `data.error.message`
3. 该错误信息以 `upstreamMessage` 为 message 抛出 `GenerationError`
4. 前端收到 `{ code: 40001, message: "The content you provided..." }`
5. 前端 `window.$message?.error(err?.message)` 展示给用户

### 10.3 改造方案

**方案：后端增加错误信息翻译（推荐）**

在 `server/src/services/generation.js` 的 `callUpstream()` 函数中，在抛出 `GenerationError` 之前，增加英文错误 → 中文翻译映射：

```js
// 常见上游错误中文翻译映射
const UPSTREAM_ERROR_MAP = [
  { pattern: /content.*(?:you provided|machine outputted).*blocked/i, message: '您提供的内容或模型生成的内容因安全策略被阻止，请调整提示词后重试' },
  { pattern: /content.*policy/i, message: '内容违反安全策略，请修改后重试' },
  { pattern: /rate limit/i, message: '请求频率过高，请稍后重试' },
  { pattern: /insufficient.*quota/i, message: 'API 额度不足，请联系管理员' },
  { pattern: /context length/i, message: '输入内容过长，请减少提示词长度' },
  // 可按需扩展
]

function translateUpstreamError(message) {
  for (const { pattern, message: zhMessage } of UPSTREAM_ERROR_MAP) {
    if (pattern.test(message)) return zhMessage
  }
  return message // 未匹配则返回原文
}
```

在 `callUpstream()` 中，对 `publicMessage` 应用翻译：
```js
const upstreamMessage = errorMessageFromUpstream(raw, `上游接口返回 ${response.status}`)
const publicMessage = translateUpstreamError(upstreamMessage)
```

**注意：** 
- 需要区分记录到日志的原始错误消息和返回给用户的公开消息
- 推荐在 `callUpstream()` 的 400+ 状态码处理分支中，将翻译后的消息作为公开消息

**或者方案：前端翻译（简单但不彻底）**

在前端 `src/utils/backend.js` 的 `friendlyNetworkError()` 函数中，或生成相关的 API 调用中增加错误消息翻译。

但后端方案更彻底，因为所有通过 API 返回的错误都会被翻译。

## 13. 注意事项

1. **余额 vs 积分使用记录：** 当前 AppHeader 显示"积分 {balance}"，改造后需要改为"积分使用记录"入口。注意区分用户页面的积分概览卡片和积分流水表格。

2. **用户组显示：** 当前 `currentUser.value?.userGroups` 是中 `fetchProfile` 从 `GET /api/auth/me` 获取的。需要确认 `/auth/me` 接口是否返回了用户组信息。

3. **公告轮询：** 前端公告采用轮询方式（30-60s），避免 SSE 的复杂度和连接管理成本。

4. **请求 ID 上下文：** 后端已有 `requestId` 中间件（`res.locals?.requestId`），新的 API 建议都传入。

5. **强制重新登录：** 修改密码成功后，前端调用 `logout()` 清除本地数据，然后跳转登录页。后端 `changePassword` 已撤销除当前会话外的所有 refresh_tokens，所以当前会话仍在，前端需要主动清除。

6. **用户组单组约束：** 约束只在新分配时生效，现有数据（已分配多组的用户）可以通过迁移脚本或逻辑清理。建议在 `assignGroup` 中添加自动清理旧组的逻辑，而不是修改数据库约束。

7. **导航栏变化：** 用户名下拉菜单中"我的画布"改为"修改密码"，但页面左上角的"我的画布"按钮（用户保留快速返回项目页的入口）仍然保留。

8. **模型价格弹窗数据源：** 可以使用现有 `GET /api/admin/models` 或计费规则 API 获取模型定价信息。
